What is MVC?

Understanding the Model-View-Controller architectural pattern

ğŸ—ï¸

MVC: Model-View-Controller

MVC is a software design pattern that separates an application into three interconnected components. This separation helps organize code, improve maintainability, and enable parallel development.

ğŸ“¦ The Three Components

ğŸ“Š

Model

The Data Layer

What is it?

The Model represents the data and business logic of your application. It directly manages the data, logic, and rules of the application.

Responsibilities:

  • Interacts with the database (Create, Read, Update, Delete)
  • Defines data structure and validation rules
  • Contains business logic and data processing
  • Notifies the view when data changes
  • Independent of the user interface

Example in Laravel:

// app/Models/User.php
class User extends Model
{
    protected $fillable = ['name', 'email', 'password'];

    public function posts()
    {
        return $this->hasMany(Post::class);
    }
}
ğŸ¨

View

The Presentation Layer

What is it?

The View is responsible for displaying data to the user. It presents the model's data in a format suitable for interaction (usually HTML).

Responsibilities:

  • Renders the user interface
  • Displays data provided by the controller
  • Contains HTML, CSS, and presentation logic
  • No business logic or database access
  • Updates when the model changes

Example in Laravel (Blade Template):

<!-- resources/views/users/profile.blade.php -->
<h1>Welcome, {{ $user->name }}</h1>

@foreach($user->posts as $post)
    <div class="post">
        <h2>{{ $post->title }}</h2>
        <p>{{ $post->content }}</p>
    </div>
@endforeach
ğŸ®

Controller

The Logic Layer

What is it?

The Controller acts as an intermediary between Model and View. It processes user input, manipulates data via the Model, and returns the appropriate View.

Responsibilities:

  • Receives user requests (HTTP requests)
  • Processes input data and validates it
  • Calls the appropriate Model methods
  • Selects the View to display
  • Acts as the middleman between Model and View

Example in Laravel:

// app/Http/Controllers/UserController.php
class UserController extends Controller
{
    public function show($id)
    {
        // Get data from Model
        $user = User::findOrFail($id);

        // Return View with data
        return view('users.profile', compact('user'));
    }
}

Understanding Mass Assignment & Security

ğŸ”’ What is Mass Assignment?

Mass Assignment allows you to create or update a model by passing an array of data directly, instead of setting each attribute individually. It's convenient but can be dangerous if not properly secured.

âœ… Mass Assignment

  'title' => 'Meetup',

Fast and convenient

âŒ Individual Assignment

Verbose and repetitive

Without Protection: Mass Assignment Vulnerability

Imagine a malicious user sending extra data in a form submission or API request. Without protection, they could modify fields they shouldn't have access to!

  "description": "New description",

The $fillable property acts as a whitelist - only the fields you explicitly list can be mass-assigned. Everything else is ignored.

        'title',
        'description',

}

Laravel automatically ignores any fields not in the $fillable array!

  • Set sensitive fields manually after validation

        return 0;  // Success exit code
